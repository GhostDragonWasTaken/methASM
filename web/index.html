MethASM

A typed, assembly-inspired language that compiles to x86-64 NASM assembly.

Statically typedC interopGarbage collectionModule systemWindows & Linux

Overview

MethASM provides explicit typing, structured control flow (if, while, for, switch), structs and enums, pointers and arrays, C interop via extern, a module system, and conservative GC for heap allocation via new. The compiler emits NASM-compatible assembly; link with the platform C runtime and any required libs (e.g. ws2_32 for networking).

Key Features

  • Explicit typing — All variables and parameters have declared types; no inference.
  • Control flow — if/else, while, for, switch with break/continue.
  • Structs & enums — Named types with methods; enums for tagged values.
  • Pointers & arrays — Bounds-aware indexing; ptr[i] for element access.
  • C interop — extern function and extern var for FFI; cstring for char*.
  • Garbage collection — new T for GC-managed heap; use malloc when GC not linked.
  • Modules — import "std/io"; stdlib under stdlib/.

Code Example

import "std/io";
import "std/net";

function main() -> int32 {
  println("Hello, MethASM!");
  var sock: int64 = socket_tcp();
  // ... bind, listen, accept ...
  return 0;
}

Standard Library

Cross-platform

  • std/io — println, puts, print_int, file I/O
  • std/mem — malloc, free, memcpy, memset
  • std/math — abs, min, max, clamp
  • std/conv — atoi, strlen, strncmp, char classification
  • std/process — exit, rand, srand

Windows (Winsock2)

  • std/net — socket, bind, listen, accept, send, recv
  • std/thread — Win32 threads, mutexes, atomics, spin locks
